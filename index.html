Contexto del Proyecto

MediRed S.A.S. operaba con tres sistemas transaccionales no integrados. El objetivo fue construir una bodega de datos corporativa que unificara esas fuentes para soportar decisiones clínicas, financieras y de gestión del talento médico.

Modelo Dimensional

Se adoptó la metodología de Kimball con un esquema estrella. Dos tablas de hechos centrales comparten un conjunto de dimensiones conformadas, permitiendo análisis cruzados entre operación y finanzas bajo la misma semántica dimensional.

Tabla de Hechos

FACT_CITAS

Grano: Una fila por cita médica agendada.

Medidas: cnt_citas · indicador_atendida · indicador_no_asistio · minutos_bloque · dis_tiempo_cita

Tabla de Hechos

FACT_FACTURAS

Grano: Una fila por factura emitida a aseguradora.

Medidas: valor_bruto · valor_copago · valor_neto · valor_glosado_total · valor_pagado_total · dias_a_pago

ETL en Apache Hop

Todos los pipelines fueron construidos en Apache Hop. Cada dimensión y tabla de hechos tiene su propio pipeline (P_DIM_* / P_FACT_*), coordinados desde un workflow maestro que controla el orden de ejecución y la dependencia entre cargas.

El pipeline integra el maestro de médicos, datos de auditoría de citas y la tabla de especialidades para enriquecer la dimensión con atributos cruzados.

medicos (CSV)→ String operations→ Replace in string ×3→ If Null→ GEN_NOMBRE_COMPLETO→ Stream lookup (audit)→ Add sequence→ Stream lookup 2 (especialidades)→ Select values ×4→ OUT_DIM_MEDICO
  • String operations + Replace in string: Normaliza mayúsculas, recorta espacios en nombres, apellidos, tipo de documento y sexo.
  • If Null: Reemplaza contactos faltantes con NO_REGISTRA en correo y celular.
  • GEN_NOMBRE_COMPLETO: User defined Java expression que concatena nombre completo sin dobles espacios, controlando ausencia de segundo nombre o apellido.
  • Stream lookup (audit_sis_citas_medicos): Enriquece con indicadores de uso del médico en el sistema de citas.
  • Add sequence: Genera clave sustituta id_dim_medico.
  • Stream lookup 2 (especialidades): Vincula atributos de especialidad a cada médico.
  • User defined Java expression 2 + Select values ×4: Calcula campos derivados, ajusta tipos y proyecta el DDL final.
especialidades (CSV)→ String operations→ If Null (NO CLASIFICADA)→ Select values→ DIM_ESPECIALIDAD
  • String operations: Recorta espacios, estandariza en mayúsculas códigos, nombres y áreas clínicas.
  • If Null: Áreas no informadas reciben valor NO CLASIFICADA como regla de negocio.
  • Select values: Proyecta columnas id, código, nombre, área, requisito de tarjeta, tiempo estándar y CUPS, ajustando tipos al DDL Oracle.
CSV file input (agendas)→ String operations→ Select values→ DIM_AGENDAS
  • Limpia y normaliza campos de texto con String operations.
  • Select values selecciona columnas, ajusta nombres a la nomenclatura del modelo y define tipos (fechas en date/time, IDs numéricos).
  • Deja la agenda consolidada y lista para análisis de capacidad y uso.
Table input (OLAP)→ LKP_DIM_PACIENTE→ LKP_DIM_AGENDAS→ LKP_DIM_MEDICOS→ LKP_DIM_SEDE→ LKP_DIM_CONTRATOS→ LKP_DIM_ESPECIALIDAD→ SEQ_ID_FACT_CITA→ Java Expr + Calculator→ Select values→ OUTPUT_FACT_CITAS
  • Encadena lookups contra todas las dimensiones, transformando claves naturales en sustitutas del DW.
  • SEQ_ID_FACT_CITA genera la PK de la tabla de hechos.
  • User defined Java expression y Calculator derivan medidas: indicadores binarios, duraciones y desvíos de tiempo.
Table input (facturas)→ Group by (métricas)→ LKP_FACT_CITAS→ LKP_DIM_ASEGURADORA→ Integrar glosas y pagos→ LKP dimensiones conformadas→ Secuencia PK→ OUTPUT_FACT_FACTURAS
  • Group by calcula métricas agregadas por factura: valor bruto, neto.
  • Integra glosas y pagos derivando valor_glosado_total, valor_pagado_total y dias_a_pago.
  • Lookups de dimensiones conformadas completan el set de FKs hacia el esquema estrella.

Calidad de Datos — Reglas Aplicadas

  • Campos de contacto vacíos → NO_REGISTRA.
  • Áreas clínicas no informadas → NO CLASIFICADA.
  • Valores de lookup sin correspondencia → miembro desconocido con id = -1.
  • Unicidad garantizada sobre num_documento en dimensiones de médicos y pacientes.
  • Fechas en formato yyyyMMdd del CSV convertidas al tipo DATE de Oracle.
  • Claves naturales de texto normalizadas a mayúsculas antes de cualquier join.

Consultas SQL Analíticas

Las consultas se diseñaron en Oracle 23c para responder las preguntas de negocio definidas en los cuatro dominios. Se utilizan funciones de ventana, CTEs y agrupaciones jerárquicas para generar indicadores de gestión.

1. Tasa de Ocupación por Sede y Especialidad

Oracle SQL — Dominio Operacional Clínico 
SELECT
  t.anio,
  t.nombre_mes,
  s.nombre_sede,
  e.nombre_especialidad,
  SUM(fc.cnt_citas)               AS total_citas,
  SUM(fc.indicador_atendida)       AS citas_atendidas,
  SUM(fc.indicador_no_asistio)     AS no_asistencias,
  ROUND(
    SUM(fc.indicador_atendida) * 100.0
    / NULLIF(SUM(fc.cnt_citas), 0), 2
  )                                AS tasa_ocupacion_pct
FROM  FACT_CITAS fc
JOIN  DIM_TIEMPO      t  ON t.id_dim_tiempo      = fc.id_dim_tiempo
JOIN  DIM_SEDE        s  ON s.id_dim_sede        = fc.id_dim_sede
JOIN  DIM_ESPECIALIDAD e  ON e.id_dim_especialidad = fc.id_dim_especialidad
GROUP BY t.anio, t.nombre_mes, s.nombre_sede, e.nombre_especialidad
ORDER BY t.anio, t.nombre_mes;

2. Facturado vs. Recaudado por Aseguradora

Oracle SQL — Dominio Financiero 
SELECT
  a.nombre_aseguradora,
  t.anio,
  t.trimestre,
  SUM(ff.valor_bruto)           AS total_facturado,
  SUM(ff.valor_pagado_total)     AS total_recaudado,
  SUM(ff.valor_glosado_total)    AS total_glosado,
  ROUND(
    SUM(ff.valor_glosado_total) * 100.0
    / NULLIF(SUM(ff.valor_bruto), 0), 2
  )                              AS tasa_glosa_pct,
  AVG(ff.dias_a_pago)           AS promedio_dias_pago
FROM  FACT_FACTURAS ff
JOIN  DIM_ASEGURADORA a  ON a.id_dim_aseguradora = ff.id_dim_aseguradora
JOIN  DIM_TIEMPO      t  ON t.id_dim_tiempo      = ff.id_dim_tiempo
WHERE ff.estado_factura != 'EMITIDA'
GROUP BY a.nombre_aseguradora, t.anio, t.trimestre
ORDER BY total_glosado DESC;

3. Ranking de Productividad Médica (RANK)

Oracle SQL — Dominio Productividad Médica · Función de Ventana RANK() 
WITH productividad AS (
  SELECT
    m.nombre_completo,
    e.nombre_especialidad,
    s.nombre_sede,
    t.anio,
    SUM(fc.cnt_citas)            AS total_citas,
    SUM(fc.indicador_atendida)   AS atendidas,
    ROUND(AVG(fc.dis_tiempo_cita),1) AS desvio_tiempo_prom,
    RANK() OVER (
      PARTITION BY t.anio
      ORDER BY SUM(fc.indicador_atendida) DESC
    )                            AS ranking_anual
  FROM  FACT_CITAS fc
  JOIN  DIM_CONTRATO_MEDICO cm ON cm.id_dim_contrato_medico = fc.id_dim_contrato_medico
  JOIN  DIM_MEDICO        m   ON m.id_dim_medico          = cm.id_dim_medico
  JOIN  DIM_ESPECIALIDAD  e   ON e.id_dim_especialidad    = fc.id_dim_especialidad
  JOIN  DIM_SEDE          s   ON s.id_dim_sede           = fc.id_dim_sede
  JOIN  DIM_TIEMPO        t   ON t.id_dim_tiempo         = fc.id_dim_tiempo
  GROUP BY m.nombre_completo, e.nombre_especialidad, s.nombre_sede, t.anio
)
SELECT * FROM productividad
WHERE  ranking_anual <= 10
ORDER BY anio, ranking_anual;

4. Validación Post-Carga — Calidad de Datos

Oracle SQL — Dominio Calidad de Datos 
-- Verificar registros huérfanos en FACT_CITAS sin dimensión de sede
SELECT COUNT(*) AS huerfanos_sede
FROM  FACT_CITAS fc
WHERE NOT EXISTS (
  SELECT 1 FROM DIM_SEDE s
  WHERE s.id_dim_sede = fc.id_dim_sede
);

-- Volumen por tabla — validación de carga completa
SELECT 'FACT_CITAS'           AS tabla, COUNT(*) FROM FACT_CITAS
UNION ALL
SELECT 'FACT_FACTURAS'        AS tabla, COUNT(*) FROM FACT_FACTURAS
UNION ALL
SELECT 'DIM_MEDICO'           AS tabla, COUNT(*) FROM DIM_MEDICO
UNION ALL
SELECT 'DIM_SEDE'             AS tabla, COUNT(*) FROM DIM_SEDE
UNION ALL
SELECT 'DIM_ESPECIALIDAD'     AS tabla, COUNT(*) FROM DIM_ESPECIALIDAD
UNION ALL
SELECT 'DIM_CONTRATO_MEDICO'  AS tabla, COUNT(*) FROM DIM_CONTRATO_MEDICO
ORDER BY 1;
