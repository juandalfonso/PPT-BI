Bodega de Datos MediRed S.A.S.

Diseño, implementación y análisis de un Data Warehouse corporativo que integra tres sistemas fuente dispersos (SIS-CITAS, SIS-FACTURA, SIS-RRHH) mediante modelado dimensional Kimball, ETL en Apache Hop, Oracle 23c y visualización en Power BI.

Contexto del Proyecto

MediRed S.A.S. operaba con tres sistemas transaccionales no integrados. El objetivo fue construir una bodega de datos corporativa que unificara esas fuentes para soportar decisiones clínicas, financieras y de gestión del talento médico.

🗄️

SIS-CITAS

Sistema Oracle para agendamiento y atención. Contiene pacientes, sedes, especialidades, médicos, agendas y citas. Base del análisis operacional de atención.

💰

SIS-FACTURA

Sistema Oracle para facturación, pagos y glosas. Gestiona aseguradoras, contratos tarifarios, facturas y recaudo. Fuente del análisis financiero.

📋

SIS-RRHH

Archivos CSV con información de médicos, contratos, sedes y especialidades. Actualización periódica desde el área de recursos humanos.

Modelo Dimensional

Se adoptó la metodología de Kimball con un esquema estrella. Dos tablas de hechos centrales comparten un conjunto de dimensiones conformadas, permitiendo análisis cruzados entre operación y finanzas bajo la misma semántica dimensional.

Tablas de Hechos

FACT_CITAS
Una fila por cita agendada
Fuente: SIS-CITAS. Mide ocupación, ausentismo y comportamiento operativo.

Medidas: cnt_citas · indicador_atendida · indicador_no_asistio · minutos_bloque · dis_tiempo_cita
FACT_FACTURAS
Una fila por factura emitida
Fuente: SIS-FACTURA + SIS-CITAS. Analiza ingresos, recaudo y glosas.

Medidas: valor_bruto · valor_copago · valor_neto · valor_glosado_total · valor_pagado_total · dias_a_pago

Dimensiones del Modelo

DIM_TIEMPO
Conformada · Compartida
Un registro por día calendario 2020–2027. Atributos: año, trimestre, mes, semana, día hábil, fin de semana.
DIM_SEDE
Conformada · Compartida
Sedes físicas de MediRed. Fuente: SIS-CITAS + sedes.csv. Atributos: código, nombre, ciudad, departamento, capacidad, gerente.
DIM_ESPECIALIDAD
Conformada · Compartida
Catálogo clínico de especialidades. Incluye área clínica, tiempo estándar de consulta y código CUPS principal.
DIM_MEDICO
Persona del médico
Atributos estables: documento, nombre completo, sexo, fecha de nacimiento, correo y teléfono.
DIM_CONTRATO_MEDICO
SCD Tipo 2 — Historial
Historia contractual: tipo de contrato, jornada, sede, especialidad, honorarios y fechas de vigencia. SCD Tipo 2 para preservar historia.
DIM_PACIENTE
Pacientes atendidos
Tipo/número de documento, nombre derivado, sexo, fecha de nacimiento y ciudad de residencia. SCD Tipo 1.
DIM_ASEGURADORA
Entidad pagadora
NIT, nombre, tipo de aseguradora y estado de vigencia. Específica del proceso financiero (FACT_FACTURAS).
DIM_AGENDAS
Agenda de consultas
Estructura de agendamiento, fechas, horas, identificadores de médico y atributos operativos de la cita programada.

ETL en Apache Hop

Todos los pipelines fueron construidos en Apache Hop. Cada dimensión y tabla de hechos tiene su propio pipeline (P_DIM_* / P_FACT_*), coordinados desde un workflow maestro que controla el orden de ejecución y la dependencia entre cargas.

P_DIM_MEDICO — Carga de dimensión de médicos

El pipeline inicia con la lectura del archivo medicos, aplica String operations y una cadena de Replace in string + Select values para normalizar mayúsculas, recortar espacios y estandarizar campos de texto (nombres, apellidos, tipo documento, sexo). Un If Null reemplaza contactos faltantes con NO_REGISTRA. El subflujo GEN_NOMBRE_COMPLETO usa una User defined Java expression para concatenar nombre completo sin dobles espacios. En paralelo, el dataset audit_sis_citas_medicos se normaliza y se une vía Stream lookup. Un Add sequence genera la clave sustituta. Finalmente, la tabla de especialidades se vincula con un segundo Stream lookup 2, varios Select values ajustan el DDL y el flujo concluye en OUT_DIM_MEDICO.

P_DIM_ESPECIALIDAD — Catálogo clínico de especialidades

El flujo lee el CSV de especialidades, aplica String operations para recortar espacios y estandarizar en mayúsculas códigos, nombres y áreas clínicas. Trata nulos con reglas de negocio (áreas no informadas → NO CLASIFICADA). Ajusta tipos y longitudes para alinearse con el DDL de DIM_ESPECIALIDAD. Proyecta las columnas (id, código, nombre, área, requisito de tarjeta, tiempo estándar, CUPS) e inserta en Oracle.

P_DIM_AGENDAS — Dimensión de agendas

El pipeline lee la estructura del archivo de agendas con un nodo CSV file input. A continuación, String operations limpia y normaliza los campos de texto. Un Select values selecciona las columnas necesarias, ajusta nombres a la nomenclatura del modelo y define tipos de datos (fechas en date/time, IDs numéricos). P_FACT_CITAS — Tabla de hechos de citas

Un Table input lee las citas del sistema OLAP. Encadena lookups contra todas las dimensiones: LKP_DIM_PACIENTE, LKP_DIM_AGENDAS, LKP_DIM_MEDICOS, LKP_DIM_SEDE, LKP_DIM_CONTRATOS_MEDICOS y LKP_DIM_ESPECIALIDAD, transformando claves naturales en sustitutas del DW. Un SEQ_ID_FACT_CITA genera la PK. Una User defined Java expression y un Calculator derivan medidas (indicadores, duraciones). Select values ajusta el DDL final. El flujo concluye en OUTPUT_FACT_CITAS.

P_FACT_FACTURAS — Tabla de hechos de facturación

El pipeline lee facturas con un Table input. Un Group by calcula métricas por factura (valor bruto, neto). Un LKP_FACT_CITAS vincula facturas con las citas registradas. En paralelo corre LKP_DIM_ASEGURADORA. Se integran glosas y pagos, calculando valor_glosado_total, valor_pagado_total y dias_a_pago. Lookups de dimensiones conformadas completan el set de FKs. La clave PK se genera por secuencia. El flujo termina en OUTPUT_FACT_FACTURAS.

Calidad de Datos — Reglas Aplicadas

Estandarización de texto
Todos los campos categóricos (nombres, códigos, especialidades) se convierten a mayúsculas y se recortan espacios sobrantes para garantizar joins correctos entre fuentes.
Tratamiento de nulos
Correos y teléfonos faltantes → NO_REGISTRA. Áreas clínicas no informadas → NO CLASIFICADA. Valores de lookup sin correspondencia → miembro desconocido con id = -1.
Deduplicación
Sort rows + Unique rows sobre num_documento en dimensiones de médicos y pacientes para garantizar unicidad de clave natural antes de insertar en Oracle.
Conversión de formatos
Fechas en formato texto yyyyMMdd del CSV convertidas al tipo DATE de Oracle. Claves naturales de texto normalizadas a mayúsculas antes de cualquier join.

Consultas SQL Analíticas

Las consultas analíticas se diseñaron en Oracle 23c para responder las preguntas de negocio definidas en los cuatro dominios. Se utilizan funciones de ventana, CTEs y agrupaciones jerárquicas para generar indicadores de gestión.

SQL · Tasa de Ocupación por Sede y Especialidad
SELECT
    t.anio,
    t.nombre_mes,
    s.nombre_sede,
    e.nombre_especialidad,
    SUM(fc.cnt_citas)              AS total_citas,
    SUM(fc.indicador_atendida)     AS citas_atendidas,
    SUM(fc.indicador_no_asistio)   AS no_asistencias,
    ROUND(
        SUM(fc.indicador_atendida) * 100.0
        / NULLIF(SUM(fc.cnt_citas), 0), 2
    ) AS tasa_ocupacion_pct
FROM   FACT_CITAS      fc
JOIN   DIM_TIEMPO      t  ON t.id_dim_tiempo      = fc.id_dim_tiempo
JOIN   DIM_SEDE        s  ON s.id_dim_sede        = fc.id_dim_sede
JOIN   DIM_ESPECIALIDAD e  ON e.id_dim_especialidad = fc.id_dim_especialidad
GROUP BY t.anio, t.nombre_mes, s.nombre_sede, e.nombre_especialidad
ORDER BY t.anio, t.nombre_mes;
SQL · Facturado vs. Recaudado por Aseguradora
SELECT
    a.nombre_aseguradora,
    t.anio,
    t.trimestre,
    SUM(ff.valor_bruto)           AS total_facturado,
    SUM(ff.valor_pagado_total)     AS total_recaudado,
    SUM(ff.valor_glosado_total)    AS total_glosado,
    ROUND(
        SUM(ff.valor_glosado_total) * 100.0
        / NULLIF(SUM(ff.valor_bruto), 0), 2
    ) AS tasa_glosa_pct,
    AVG(ff.dias_a_pago)            AS promedio_dias_pago
FROM   FACT_FACTURAS    ff
JOIN   DIM_ASEGURADORA  a  ON a.id_dim_aseguradora = ff.id_dim_aseguradora
JOIN   DIM_TIEMPO       t  ON t.id_dim_tiempo      = ff.id_dim_tiempo
WHERE  ff.estado_factura != 'EMITIDA'
GROUP BY a.nombre_aseguradora, t.anio, t.trimestre
ORDER BY total_glosado DESC;
SQL · Productividad Médica — Top 10 por Citas Atendidas
WITH productividad AS (
    SELECT
        m.nombre_completo,
        e.nombre_especialidad,
        s.nombre_sede,
        t.anio,
        SUM(fc.cnt_citas)           AS total_citas,
        SUM(fc.indicador_atendida)  AS atendidas,
        ROUND(AVG(fc.dis_tiempo_cita),1) AS desvio_tiempo_prom,
        RANK() OVER (
            PARTITION BY t.anio
            ORDER BY SUM(fc.indicador_atendida) DESC
        ) AS ranking_anual
    FROM   FACT_CITAS          fc
    JOIN   DIM_CONTRATO_MEDICO cm ON cm.id_dim_contrato_medico = fc.id_dim_contrato_medico
    JOIN   DIM_MEDICO          m  ON m.id_dim_medico          = cm.id_dim_medico
    JOIN   DIM_ESPECIALIDAD    e  ON e.id_dim_especialidad    = fc.id_dim_especialidad
    JOIN   DIM_SEDE            s  ON s.id_dim_sede            = fc.id_dim_sede
    JOIN   DIM_TIEMPO          t  ON t.id_dim_tiempo          = fc.id_dim_tiempo
    GROUP BY m.nombre_completo, e.nombre_especialidad, s.nombre_sede, t.anio
)
SELECT * FROM productividad
WHERE  ranking_anual <= 10
ORDER BY anio, ranking_anual;
SQL · Validación Post-Carga — Integridad Referencial
-- Verificar registros huérfanos en FACT_CITAS sin dimensión de sede
SELECT COUNT(*) AS huerfanos_sede
FROM  FACT_CITAS fc
WHERE NOT EXISTS (
    SELECT 1 FROM DIM_SEDE s
    WHERE s.id_dim_sede = fc.id_dim_sede
);

SELECT 'FACT_CITAS'         AS tabla, COUNT(*) FROM FACT_CITAS
